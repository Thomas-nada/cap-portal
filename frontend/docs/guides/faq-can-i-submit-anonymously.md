# Can I submit a CAP anonymously?

You can submit pseudonymously. Fully anonymous submission — with no identity attached at any level — is not possible through the standard process, because GitHub Issues are tied to a GitHub account.

---

## What "pseudonymous" means here

You can create a GitHub account under any username and submit proposals under that handle. The public record will show your GitHub username, not your real name. The CAP process has no identity verification requirement — no KYC, no real-name policy, no verification against on-chain identity.

Many community members participate under handles rather than legal names.

---

## What appears on the public record

When you submit a CAP, two things are publicly associated with it:

1. **Your GitHub username** — the account that opened the issue
2. **The Authors field** in the YAML frontmatter — whatever name or handle you provide here

You control both. You can use the same pseudonym for both, a different one, or leave the Authors field as a handle only.

---

## Practical considerations

If you want to be reachable for feedback and deliberation, your GitHub username serves as the contact point. Community members and Editors will tag you in comments using that handle.

If your proposal is accepted and you want attribution, the Authors field in the frontmatter is the formal record. Choose a handle you are comfortable being associated with that proposal long-term — it will be in the repository permanently.

---

## On-chain anonymity is a separate question

If you later submit a governance action on-chain yourself, on-chain identity is governed by the Cardano ledger, not the CAP process. The CAP Portal and GitHub process do not interact with on-chain identity.
