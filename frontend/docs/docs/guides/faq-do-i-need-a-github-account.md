# Do I need a GitHub account?

It depends on what you want to do.

---

## You do NOT need a GitHub account to:

- Read any proposal — all issues are publicly visible
- Read the Constitution in the [Constitution viewer](/#/constitution)
- Browse the CAP Portal, proposals list, and Learn Hub

---

## You DO need a GitHub account to:

- **Submit a CAP or CIS** — proposals are submitted as GitHub Issues
- **Comment on a proposal** — participating in deliberation requires a GitHub account
- **React to a proposal** — the 👍 👎 reactions that signal community sentiment require sign-in
- **Subscribe to notifications** — following a specific issue requires sign-in
- **Watch the repository** — to receive notifications for all activity

---

## Creating an account

GitHub accounts are free. Create one at [github.com](https://github.com). You do not need to provide payment information or verify your identity beyond a valid email address.

---

## Pseudonymous participation

You do not need to submit under your real name. GitHub allows you to choose any username, and the CAP process has no identity verification requirement. Many community members participate pseudonymously.

If you are submitting a CAP and want credit for authorship, include the name or handle you want associated with the proposal in the Authors field of the frontmatter. Whatever you put there becomes the public record.

---

## The CAP Portal and authentication

The [CAP Portal](https://thomas-nada.github.io/CAP) can be used fully without authentication. The **Amendment Wizard** — the guided proposal drafting tool — is available to all users.

When you reach the submission step, the Wizard will direct you to the GitHub Issues form to complete your submission, which is where the account requirement applies.
